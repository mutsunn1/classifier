# 笔记分类 Agent 技能模式

## 概述
本文档总结了表现良好的笔记分类 Agent 的执行流程和最佳实践，旨在降低不同参数模型在同一系统中的执行效果差异。

## 核心执行流程

### 1. 资产盘点阶段
**目标**: 发现所有待处理的笔记文件
**最佳实践**:
- 调用 `list_files` 工具扫描源目录
- 记录文件数量、文件名和路径
- 验证文件可读性（可选）

### 2. 内容分析阶段
**目标**: 理解每个笔记文件的内容主题
**最佳实践**:
- 对每个文件调用 `read_file` 读取内容
- 提取关键信息：标题、摘要、分类标签
- 生成一句话摘要（用于后续分类）

### 3. 分类规划阶段
**目标**: 确定分类体系和文件夹结构
**最佳实践**:
- 分析所有笔记的主题分布
- 设计互斥且合理的分类体系（如：数学、物理、化学、计算机科学等）
- 避免过于宽泛的分类（如"学习资料"）
- 避免为每个文件单独创建文件夹

### 4. 目录创建阶段
**目标**: 在目标目录创建分类文件夹
**最佳实践**:
- 调用 `create_directory` 为每个分类创建文件夹
- 确保文件夹名称简洁、明确
- 创建"未分类"文件夹用于无法分类的内容
- 验证文件夹创建成功

### 5. 文件移动阶段
**目标**: 将笔记文件移动到对应的分类文件夹
**最佳实践**:
- 对每个文件调用 `move_file` 工具
- 使用正确的源路径和目标路径
- 处理文件名冲突（如重命名或跳过）
- 记录移动结果（成功/失败）

### 6. 索引创建阶段
**目标**: 生成导航索引文件
**最佳实践**:
- 调用 `write_file` 创建 `README.md` 文件
- 使用正确的 Markdown 相对路径格式
- 包含清晰的目录结构
- 为每个笔记添加一句话摘要
- 验证链接可用性

## 错误处理策略

### 1. 文件读取失败
- 记录错误并跳过该文件
- 将无法读取的文件放入"未分类"文件夹

### 2. 分类不确定
- 当内容模糊或跨多个分类时，放入"未分类"文件夹
- 避免强行分类导致错误

### 3. 路径问题
- 正确处理特殊字符和空格
- 使用相对路径而非绝对路径
- 验证路径存在性

### 4. 文件冲突
- 检测目标文件是否已存在
- 采用重命名策略（如添加后缀）
- 记录冲突处理结果

## 性能优化建议

### 1. 批量操作
- 批量读取文件内容，减少 I/O 操作
- 批量创建目录，避免重复检查
- 批量移动文件，提高效率

### 2. 内存管理
- 逐文件处理大文件，避免内存溢出
- 及时释放不再需要的数据

### 3. 进度跟踪
- 显示处理进度和状态
- 记录成功和失败的数量

## 质量保证

### 1. 分类准确性检查
- 抽查分类结果，验证文件与文件夹的匹配度
- 确保分类体系合理且互斥

### 2. 索引文件验证
- 测试所有 Markdown 链接的可用性
- 检查索引文件的格式和完整性

### 3. 完整性验证
- 验证源文件数量与目标文件数量一致
- 确保没有文件丢失或重复

## 示例执行顺序

1. `list_files("note/")` → 获取所有笔记文件
2. 对每个文件: `read_file("note/笔记1.txt")` → 分析内容
3. `create_directory("organized_notes/数学/")` → 创建分类文件夹
4. `move_file("note/笔记1.txt", "organized_notes/数学/笔记1.txt")` → 移动文件
5. `write_file("organized_notes/README.md", "...")` → 创建索引

## 注意事项

1. **上下文管理**: 避免在单个提示中包含过多文件内容，可能导致上下文窗口溢出
2. **工具调用顺序**: 遵循"先读后写，先建后移"的原则
3. **路径处理**: 始终使用正确的路径分隔符和相对路径
4. **错误恢复**: 设计容错机制，避免单点失败导致整个任务中止
